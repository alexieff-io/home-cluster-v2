# Coroot (Community Edition) Implementation Plan

> **For agentic workers:** REQUIRED SUB-SKILL: Use superpowers:subagent-driven-development (recommended) or superpowers:executing-plans to implement this plan task-by-task. Steps use checkbox (`- [ ]`) syntax for tracking.

**Goal:** Deploy Coroot Community Edition to the cluster as a GitOps-managed app (operator + `Coroot` CR) with bundled ClickHouse/Prometheus and an internal HTTPRoute.

**Architecture:** A `coroot-operator` HelmRelease installs the `coroot.com` CRDs and controller; a plain `Coroot` custom resource (Community Edition) tells the operator to deploy the Coroot server + bundled ClickHouse + Prometheus + node-agent + cluster-agent. A standalone HTTPRoute exposes the Coroot service internally. Mirrors the existing `monitoring/victoria-metrics-operator` → `VMSingle` operator pattern.

**Tech Stack:** Flux CD, Helm (`coroot-operator` chart `0.x`), Gateway API (Envoy), Longhorn storage, kubeconform/flux-local validation.

**Spec:** `docs/superpowers/specs/2026-06-05-coroot-design.md`

**Branch:** Work happens on `feat/coroot` (already created).

## File structure

```
kubernetes/apps/coroot/
├── kustomization.yaml              # namespace + both app ks.yaml entries
├── namespace.yaml                  # ns "coroot" + goldilocks label
├── coroot-operator/
│   ├── ks.yaml                     # Flux Kustomization, wait: true
│   └── app/
│       ├── kustomization.yaml
│       ├── helmrepository.yaml     # https://coroot.github.io/helm-charts
│       └── helmrelease.yaml        # chart coroot-operator 0.x
└── coroot/
    ├── ks.yaml                     # Flux Kustomization, dependsOn coroot-operator, substituteFrom cluster-secrets
    └── app/
        ├── kustomization.yaml
        ├── coroot.yaml             # Coroot CR (Community Edition)
        └── httproute.yaml          # coroot.${SECRET_DOMAIN} → envoy-internal → coroot:8080
```

All Flux Kustomizations live in the `flux-system` namespace (the `coroot/kustomization.yaml` does NOT set a top-level `namespace:`, so no rewrite happens — unlike `kube-system/`). Flux's root `cluster-apps` Kustomization (`path: ./kubernetes/apps`, recursive autodetect) discovers the new folder automatically; no other file needs editing.

```yaml

  name: coroot-operator
  namespace: flux-system

```

- [ ] **Step 3: Create the HelmRelease**

`kubernetes/apps/coroot/coroot-operator/app/helmrelease.yaml`:

```yaml
---
apiVersion: helm.toolkit.fluxcd.io/v2
kind: HelmRelease
metadata:
  name: coroot-operator
spec:
  chart:
    spec:
      chart: coroot-operator
      version: 0.x
      sourceRef:
        kind: HelmRepository
        name: coroot
  interval: 1h
  values:
    resources:
      requests:
        cpu: 100m
        memory: 64Mi
      limits:
        memory: 128Mi

      allowPrivilegeEscalation: false
      readOnlyRootFilesystem: true
      runAsNonRoot: true

        drop: ["ALL"]
```

> `version: 0.x` floats the minor (chart is `0.9.6`, appVersion `1.9.5`) so Renovate manages it — matches the `victoria-metrics-operator` `0.x` convention. The root `cluster-apps` patch already injects `install.crds: CreateReplace` / `upgrade.crds: CreateReplace`, so CRDs upgrade in place.

## Task 3: coroot CR + HTTPRoute app

**Files:**
- Create: `kubernetes/apps/coroot/coroot/ks.yaml`
- Create: `kubernetes/apps/coroot/coroot/app/kustomization.yaml`
- Create: `kubernetes/apps/coroot/coroot/app/coroot.yaml`
- Create: `kubernetes/apps/coroot/coroot/app/httproute.yaml`

- [ ] **Step 1: Create the Flux Kustomization**

`kubernetes/apps/coroot/coroot/ks.yaml`:

```yaml
---
apiVersion: kustomize.toolkit.fluxcd.io/v1
kind: Kustomization
metadata:
  name: coroot
  namespace: flux-system
spec:
  interval: 1h
  path: ./kubernetes/apps/coroot/coroot/app
  postBuild:
    substituteFrom:
      - name: cluster-secrets
        kind: Secret
  prune: true
  sourceRef:
    kind: GitRepository
    name: flux-system
    namespace: flux-system
  targetNamespace: coroot
  wait: false
  dependsOn:
    - name: coroot-operator
      namespace: flux-system
```

> `postBuild.substituteFrom: cluster-secrets` enables `${SECRET_DOMAIN}` in the HTTPRoute. `dependsOn: coroot-operator` (explicit `namespace: flux-system`) ensures the CRDs/operator are Ready first.

- [ ] **Step 2: Create the Coroot custom resource**

`kubernetes/apps/coroot/coroot/app/coroot.yaml`:

```yaml
---
apiVersion: coroot.com/v1
kind: Coroot
metadata:
  name: coroot
spec:
  communityEdition:
    image:
      name: ghcr.io/coroot/coroot:1.22.0
      pullPolicy: IfNotPresent
  storage:
    size: 10Gi
    className: longhorn
  prometheus:
    retention: 2d
    storage:
      size: 10Gi
      className: longhorn
  clickhouse:
    shards: 1
    replicas: 1
    storage:
      size: 10Gi
      className: longhorn
  nodeAgent:
    resources:
      requests:
        cpu: 100m
        memory: 200Mi
      limits:
        cpu: 500m
        memory: 1Gi
  clusterAgent: {}
```

> Field names verified against `coroot/coroot-operator` `api/v1` (`StorageSpec.className`/`size`, `PrometheusSpec.retention`, `ClickhouseSpec.shards`/`replicas`). Image tag `1.22.0` confirmed present on `ghcr.io/coroot/coroot` (note: no `v` prefix on the registry).

- [ ] **Step 3: Create the HTTPRoute**

`kubernetes/apps/coroot/coroot/app/httproute.yaml`:

```yaml
---
apiVersion: gateway.networking.k8s.io/v1
kind: HTTPRoute
metadata:
  name: coroot
spec:
  hostnames: ["coroot.${SECRET_DOMAIN}"]
  parentRefs:
    - name: envoy-internal
      namespace: network
      sectionName: https
  rules:
    - backendRefs:
        - name: coroot
          port: 8080
```

> Backend `coroot:8080` — the operator names the Service after the CR (`coroot`) and the default service port is `8080` (`ServiceSpec.Port` default). **Confirm at deploy time** (Task 6 Step 4); if the operator names it differently, update `backendRefs[0].name` here.

- [ ] **Step 4: Create the app kustomization**

`kubernetes/apps/coroot/coroot/app/kustomization.yaml`:

```yaml
---
apiVersion: kustomize.config.k8s.io/v1beta1
kind: Kustomization
resources:
  - ./coroot.yaml
  - ./httproute.yaml
```

- [ ] **Step 5: Build the coroot app locally**

Run: `kustomize build kubernetes/apps/coroot/coroot/app`
Expected: clean render of the Coroot CR + HTTPRoute (note `${SECRET_DOMAIN}` is unresolved here — Flux substitutes it at apply time; that is expected in a raw `kustomize build`).
